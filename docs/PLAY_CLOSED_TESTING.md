# Google Play：封闭式测试 — 详细步骤与清单

> **说明**：我无法替你「注册真实邮箱」或代你登录 Google。下面提供 **登记表模板** 与 **控制台操作步骤**；测试邮箱需你本人逐个申请并加入列表。

---

## 一、封闭测试要达成什么效果？

| 目标 | 说明 |
|------|------|
| **安装与启动** | 用**真实设备**从 Play 提供的**测试链接**安装，能打开、不闪退。 |
| **核心路径** | 登录/注册、生成报告、（若测付费）订阅沙盒或恢复购买。 |
| **稳定性** | 连续使用几天，观察是否崩溃、卡顿、异常耗电（Play 控制台可看 **Android vitals**）。 |
| **与生产接近** | 使用与计划上架**相同或更高**的 `versionCode` 的 AAB，配置与正式一致（API 域名、签名轨道最终要一致策略）。 |

建议 **至少跑 3～7 天**（非 Google 强制，是工程上容易暴露问题的窗口）；团队极小可以 **3 天** 后若无大坑再推进生产。

## 三、Play Console 操作步骤（按界面可能略有出入）

以下路径以 **Google Play Console** 英文界面为参考；中文菜单名称可能略有差异。

1. 打开 [Google Play Console](https://play.google.com/console)，选择应用 **`com.crickettechnology.reportifyai`**（或你的包名）。  
2. 左侧进入 **Testing（测试）** → **Closed testing（封闭测试）**。  
3. 若尚无轨道：**Create track（创建轨道）** → 命名如 `closed-alpha`。  
4. **Testers（测试人员）**：  
   - 选择 **Email lists（电子邮件列表）** 或 **Google Groups**；  
   - **Create email list（创建电子邮件列表）**，把上表邮箱 **逐条粘贴/添加** 保存。  
5. **Releases（版本）**：在封闭测试轨道里 **Create new release（创建新版本）**，上传 **AAB**（与内部测试同一签名、**versionCode 递增**）。  
6. 保存并 **Review → Start rollout to Closed testing（发布到封闭测试）**。  
7. 在 **Testers** 页面复制 **Opt-in URL（选择加入链接）**，用**未在列表外的账号**打开会装不了；用**列表内账号**在手机上打开链接 → **Become a tester（成为测试员）** → Play 商店安装/更新。

**注意**：封闭测试的 App 需 **使用该轨道发布的版本**；内部测试与封闭测试是 **不同轨道**，版本要分别上传/提升。

## 四、在手机上怎么测？测什么？

1. **用列表内 Gmail 登录手机上的 Google Play**（或添加该 Google 账号）。  
2. 打开 **Opt-in 链接** → 接受测试 → 在 Play 里安装/更新 App。  
3. 逐项验证：  
   - 冷启动、杀进程再开；  
   - 登录、生成报告；  
   - 若测订阅：沙盒购买、恢复购买、与你们 **api** 侧套餐是否一致；  
   - 弱网/飞行模式恢复（可选）。  
4. 每天扫一眼 Play Console **Quality（质量）** → **Android vitals** / **Crashes（崩溃）**。

---

## 五、封闭测试完成后 → 提交「上线 / 正式版」

**流程上**：封闭测试轨道验证 OK 后，在控制台将 **同一 AAB（或已修复的新 AAB）** 提升到 **Production（正式版）** 轨道，填写 **发布说明**，提交 **Google 审核**。

**不是**「封闭测试结束就自动上架」；需要你在 **Production** 里 **手动创建发行版并提交审核**。

---

## 六、上架/审核常见材料清单（对照是否已有）

| 材料 | 说明 |
|------|------|
| **隐私政策 URL** | 公网可访问；内容与 App 实际收集数据一致。 |
| **应用内容 / 目标受众** | 是否面向儿童、新闻类声明等（按向导填）。 |
| **数据安全表单（Data safety）** | 收集哪些数据、是否加密、是否分享第三方。 |
| **权限说明** | 清单里每个敏感权限与功能对应。 |
| **订阅** | 若在 Play 售订阅：商品已配置、**基础方案/优惠**与客户端一致。 |
| **商店素材** | 简短说明、完整说明、图标、截图、功能图（各尺寸按控制台要求）。 |
| **内容分级** | 问卷作答。 |
| **联系邮箱** | 支持/开发者联系。 |

你们若 **网站隐私页、图标、截图** 已有，多半只需 **核对与 App 行为一致** 并补全 **数据安全** 问卷。

---

## 七、时间线建议（小团队）

| 阶段 | 建议时长 |
|------|----------|
| 封闭测试实机使用 | **3～7 天** |
| 修崩溃/计费问题 | 视情况 1～若干天 |
| 提审正式版 | 审核 **数小时～数日**（因应用而异） |

若你希望「尽量少步骤」：**封闭测试 3 天无大崩溃 → 提生产审核**，是务实节奏。
